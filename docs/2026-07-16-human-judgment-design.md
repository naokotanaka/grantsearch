# 人間の判定（👍関係ある／👎関係ない）設計書

日付: 2026-07-16
状態: 承認済み（実装へ）

## 目的

レポート（特に🔎新着・発見）の各助成金について、「ウチに関係ある／ない」を
人間がワンタップでシステムに教えられるようにする。判定は再検索で消えず、
表示のフィルタとAIの判断材料の両方に使う。

## データモデル

- `Grant.humanJudgment`: `'' | '関係あり' | '関係ない'`（既定 `''`）
- `memo`・`manualUrl` と同じ「人間の入力」扱い：
  - `upsertGrant` の ON CONFLICT 更新対象から外す
  - 専用の `updateHumanJudgment(db, id, judgment)` だけが書き換える
- SQLite 列は `human_judgment TEXT DEFAULT ''`（既存DBには ALTER TABLE で追加）

## レポートUI

- 各行の ✏📎 の並びに 👍👎 ボタンを追加（全セクション）
- 👎 → `POST api/judgment {id, judgment:'関係ない'}` → レポート再生成。
  行は通常セクションから消え、最下部の折りたたみ
  「👎 関係ないと判定したもの」に入る。「戻す」ボタン（judgment:''）で取り消し可
- 👍 → `POST api/judgment {id, judgment:'関係あり'}`。行に👍バッジが付いて残る

## 検索パイプラインへの効果

- **👎 関係ない**:
  - AI読み取り（enrich）をスキップ（読み取り枠150件の節約）
  - 最終リストから外し `hidden=1`。ただし判定・行自体はDBに残す
  - レポートの折りたたみセクションは `human_judgment='関係ない'` をDBから直接読む
- **👍 関係あり**:
  - AIが「対象外」と判断しても消さない（manualUrl登録時と同じ降格保護：要確認に落とすだけ）
  - known-grants-checker と同様に毎週公式ページ（manualUrl 優先）をフェッチし、
    募集開始を検知したら `募集中` に自動昇格（発掘→追跡への昇格）

## AIへのフィードバック

- AI読み取りのプロンプトに、人間の判定履歴を渡す：
  - 「過去に『関係ない』と判定された助成金名」（最大30件）
  - 「過去に『関係あり』と判定された助成金名」（最大30件）
- 名前だけなのでトークンコストは微小。似た系統の新規発掘品の判断精度を上げる

## API

- `POST /api/judgment` … body `{id, judgment}`。judgment は
  `'関係あり' | '関係ない' | ''` のみ受理。保存後にレポート再生成。
  memo/manual-url と同じく相対パスで呼ぶ（nginx のプレフィックス除去対応）

## 代替案と不採用理由

- メモ欄に「関係ない」と書いて流用 → メモ本来の用途と混ざり誤爆しやすい
- 表示フィルタのみ（AIに渡さない）→「AIに伝えたい」という要望を満たさない
